# Changelog

All notable changes to this project will be documented in this file.

The format is inspired by Keep a Changelog, and this project follows semantic versioning as closely as practical during early releases.

## [0.1.0] - 2026-04-07

### Added
- First tagged Julia package release of `TFMisfitGOF`.
- Julia package structure under `src/` with a loadable `TFMisfitGOF` module.
- CLI commands for `prepare`, `run`, `plot`, `pipeline`, and `validate`.
- Python preprocessing helper for generating `HF_TF-MISFIT_GOF` input files from CSV data.
- Python plotting helper for figures generated from run outputs.
- Canonical `results.h5` output for plotting and downstream inspection.
- Configurable legacy output modes for backward compatibility:
  - `h5`
  - `summary`
  - `full`
- Reproducible example workflows under `examples/`.
- Automated tests under `test/`.
- CI scaffolding under `.github/workflows/`.
- Citation metadata via `CITATION.cff`.

### Changed
- Refactored the project from a monolithic Julia script into a package-oriented layout.
- Established the Julia CLI as the recommended interface for running the workflow.
- Made `results.h5` the canonical output artifact for modern workflows.
- Kept legacy ASCII `.DAT` outputs as compatibility artifacts rather than the primary run product.
- Improved plotting workflow portability with CLI-controlled plotting options.

### Fixed
- Wired the split numerical implementation into the package entrypoints.
- Restored end-to-end example validation through the CLI.
- Fixed CLI required-option parsing for commands such as `validate`.
- Fixed relative-path handling when validating example runs.
- Fixed HDF5 writing issues caused by non-materialized transposed arrays.
- Aligned the numerical driver with the actual namelist keys generated by the preprocessing script.

### Notes
- `results.h5` is the canonical output format going forward.
- Legacy `.DAT` files remain available for backward compatibility with older workflows, external scripts, and existing users.
- Declared Julia 1.9+ support in `Project.toml`; Julia 1.10+ is recommended in the user-facing documentation.
- The recommended modern workflow is:
  1. `prepare`
  2. `run` or `pipeline`
  3. `plot`
  4. `validate` for bundled examples when needed
